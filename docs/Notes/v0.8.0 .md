اقتراح معماري للمراحل القادمة

بعد مراجعة المشروع، أرى أن MosqueDashboardResponse سيستمر في النمو مع كل وحدة جديدة (متطوعون، حملات، مستودع، تعليم...). لذلك أقترح أن يكون هذا آخر إصدار يعتمد على حقول مسطحة (Flat Fields).

في Milestone 1.0 يمكن إعادة تصميمه ليصبح أكثر تنظيمًا، مثل:

public class MosqueDashboardResponse {

    private MembershipStatistics membership;

    private PositionStatistics positions;

    private DonationStatistics donations;

    private VolunteerStatistics volunteers;

    private InventoryStatistics inventory;

}

في المرحلة الحالية لا بأس بذلك، لكن في Milestone 0.9 أو 1.0 أنصح بإضافة استعلامات مثل:

long countByMosque(Mosque mosque);

long countByMosqueAndStatus(
        Mosque mosque,
        DonationStatus status);

في DonationRepository.

بعدها يصبح المزود أكثر كفاءة:

dashboard.setTotalDonations(
        donationRepository.countByMosque(mosque));

dashboard.setPendingDonations(
        donationRepository.countByMosqueAndStatus(
                mosque,
                DonationStatus.PENDING));

وهكذا لبقية الحالات، دون تحميل جميع التبرعات إلى الذاكرة. هذا الأسلوب سيكون أكثر أداءً عندما يكبر عدد التبرعات في المنصة.
